### 开发目标

[BotBattle项目完整代码](https://github.com/xiu1zi3/Bot-Battle) 主目录`backendcloud`下有 3 个微服务项目`backend`、`botrunningsystem`、`matchingsystem`

```shell
 │
 ├─backend
 │
 ├─botrunningsystem
 │
 └─matchingsystem
```

将以上 3 个项目打包成`.jar`文件并传到服务器运行：

`java -jar backend-0.0.1-SNAPSHOT.jar`  

`java -jar matchingsystem-0.0.1-SNAPSHOT.jar`  

`java -jar botrunningsystem-0.0.1-SNAPSHOT.jar`

### 问题一

#### spring-boot-maven-plugin 版本不识别的问题

在打包 maven 项目时报错：

```shell
org/springframework/boot/maven/RepackageMojo has been compiled by a more recent version of the Java Runtime (class file version 61.0), this version of the Java Runtime only recognizes class file versions up to 52.0
```

#### 解决：添加版本号

对应的模块的`pom.xml`中的`<build>`标签里，加上合适的版本号

```xml
<groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-maven-plugin</artifactId>
    <!--在此处加上版本号-->
```

 [我真的菜的要死了](https://www.acwing.com/user/myspace/index/109718/) 建议，直接用和 springboot 相同版本号。而我使用的 springboot 版本号是`3.2.3`，如此尝试并不奏效

根据 [西凉的悲伤](https://huaweicloud.csdn.net/63aab976b878a54545947e39.html/) 的博客介绍，使用 Xtools 打包，该版本号`<version>2.6.0</version>`可行！

### 问题二

#### 打包问题

在服务器上运行`java -jar test-0.0.1-SNAPSHOT.jar`，
报错：

```shell
no main manifest attribute, in test-0.0.1-SNAPSHOT.jar
```

#### 解决：核验主类并修改标签

发生原因是 java 找不到主类

> 一般情况下，java 打包成 jar 包需要在`MANIFEST.MF`中指定 Main-Class 项，以便运行`java -jar xxx.jar`时找到对应的主类。

将`test-0.0.1-SNAPSHOT.jar`解压，
查看 META-INF 目录下的`MANIFEST.MF`文件，理应包含`Start-class`项。

[Makisekurisu_4](https://www.acwing.com/user/myspace/index/80884/) 指出，应在主目录`backendcloud`（也就是总项目）的`pom.xml`文件中如下修改

```xml
                <configuration>
                    <mainClass>com.kob.backendcloud.BackendcloudApplication</mainClass>
                    <skip>true</skip>  //这个标签被标记成true之后,子项目打包的时候全部信息都没有了.
                    //解决方案: 删除这个标签,或者是改成false
                </configuration>
```

`<skip>`改成`true`后，果然 META-INF 目录下的`MANIFEST.MF`文件包含`Start-class`项

服务器成功执行以下三条命令，问题解决！

`java -jar backend-0.0.1-SNAPSHOT.jar`  

`java -jar matchingsystem-0.0.1-SNAPSHOT.jar`  

`java -jar botrunningsystem-0.0.1-SNAPSHOT.jar`
